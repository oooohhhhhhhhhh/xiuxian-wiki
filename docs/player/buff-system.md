# Buff系统

Buff系统允许玩家通过使用物品获得临时属性加成，这些加成在一段时间后自动过期并扣除相应属性。

## Buff属性

| 属性类型 | 说明 | 最大值限制 |
|----------|------|------------|
| 攻击加成 | 增加角色攻击力 | +200 |
| 防御加成 | 增加角色防御力 | +200 |
| 速度加成 | 增加角色速度 | +100 |
| 灵力加成 | 增加角色灵力 | +100 |

## Buff机制

### Buff叠加

多个同类 Buff 可以叠加，但有上限：

| 属性 | 叠加上限 |
|------|----------|
| 攻击 | +200 |
| 防御 | +200 |
| 速度 | +100 |
| 灵力 | +100 |

### Buff过期

Buff 会在指定时间后自动过期：

- **检查频率**：每 10 秒检查一次
- **过期处理**：自动移除 Buff，扣除相应属性
- **通知方式**：通过 WebSocket 推送过期消息

### Buff优先级

当多个 Buff 叠加超过上限时，按优先级保留：

1. 剩余时间最长的 Buff
2. 属性值最高的 Buff

## Buff 效果计算

### 战斗中的 Buff 效果

```
实际攻击 = 基础攻击 × (1 + Buff攻击加成%) + Buff攻击数值
实际防御 = 基础防御 × (1 + Buff防御加成%) + Buff防御数值
实际速度 = 基础速度 × (1 + Buff速度加成%) + Buff速度数值
实际灵力 = 基础灵力 × (1 + Buff灵力加成%) + Buff灵力数值
```

### 示例计算

玩家基础属性：攻击 100，防御 80，速度 50，灵力 30

使用战神丹（攻击+50，持续3分钟）和金刚护符（防御+50，持续3分钟）：

- 实际攻击 = 100 + 50 = 150
- 实际防御 = 80 + 50 = 130
- 实际速度 = 50
- 实际灵力 = 30

## Buff 管理

### QQ 指令

| 指令 | 说明 |
|------|------|
| `/buff` | 查看当前激活的 Buff |

### REST API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/game/buff` | 查看当前激活的 Buff |

### WebSocket 消息

| 消息类型 | 说明 |
|----------|------|
| `buff` | Buff 添加/过期通知 |
